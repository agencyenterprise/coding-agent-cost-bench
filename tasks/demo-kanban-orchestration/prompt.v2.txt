## Task
Build a single-board kanban app (Trello-like) with Next.js (App Router, TypeScript) in the current
workspace root — do not create a nested subdirectory. A board has columns (e.g. "To Do", "In
Progress", "Done"), each holding cards with a title and description. Support creating, editing,
deleting, and moving a card between and within columns, plus adding and removing columns.
Drag-and-drop is a nice touch but use any approach you like — don't reach for a heavy library.

## Success criteria
- `npm run build` completes with no errors (it type-checks under TypeScript too).
- `npx vitest run` passes, with at least one test covering the store's move-card logic.
- The app serves a seeded board: `/api/health` returns 200, and `/api/board` returns the full board
  as JSON with 3 named columns and 6 titled cards.

## Scope
- Back the board with Next.js API route handlers over an in-memory store, seeded deterministically
  at boot with 3 columns and 6 cards. Design the route shapes (CRUD + move) yourself.
- Keep it minimal — just enough dependencies and UI for a working board. No auth, no database, no
  external services, no API keys, no ESLint/coverage/CI wiring, no README.
- Componentize sensibly; don't dump everything into one file.

## Environment
- Already scaffolded: `create-next-app` (App Router, TypeScript, Tailwind) has run and dependencies
  (including Vitest) are installed. Do NOT re-run `create-next-app` or reinstall — build in place.
- Install only anything additional you actually need.

## Before finishing
- Run `npx vitest run` and `npm run build` once, and fix anything that fails.
- You do NOT need to boot the server or curl it yourself — just make sure the API routes and the
  3-column / 6-card seed are in place.
